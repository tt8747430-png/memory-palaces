# P1 — Decks Area Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild the entire Decks feature-area View layer in React 19 + shadcn (Base UI) at parity with
`main`, on top of the framework-agnostic core P0 restored — split into three independently-green
sub-phases (P1a shell substrate → P1b Library cluster → P1c Detail/Questions/Settings/Content).

**Architecture:** The domain, models, commands, stores, and `Repository<T>` seam are already ported and
green (P0). This plan rebuilds only the View: app chrome, the overlay/Drawer contract, and the decks
pages. `main` is the **behavioural parity bar** (worktree at `../mindscape-ref`); its View is copied
across a fixed set of mechanical transforms (§ Port Transform Table), not reinvented. Earned
ViewModels become hooks (A.7 of the master plan); everything else reads stores directly.

**Tech Stack:** React 19.2 · React Router 8 (data mode) · shadcn/ui `base` (Base UI) · Tailwind v4 ·
`@dnd-kit` · `motion` · `sonner` · `lucide-react` · Vitest 4 + Testing Library + axe-core.

**Parent plan:** [`2026-07-16-return-to-react.md`](./2026-07-16-return-to-react.md) (P0 complete at commit
`ebf1088`). This document is P1's own plan, as that plan's Part C requires each phase to earn.

**Source spec / decisions:** Parent plan Parts A (consolidated decisions), B (target structure). ADRs
0010 (Drawer contract), 0011 (rem tokens), 0012 (route transitions).

## Global Constraints

Inherits the parent plan's Global Constraints **verbatim** (zero-legacy code, `version: 0` schemas, the
`shared/` boundary rule, commands-for-all-writes, `start()` only at the composition root, semantic
tokens only, earned ViewModels, rem token scale, TS strict, Prettier no-semi/single-quote/width-100).
P1-specific additions:

- **`main` is the parity bar, not the Angular tree.** Where they disagree, `main` wins; a deviation
  worth keeping is raised as a decision, never smuggled in as a port. Run `../mindscape-ref` (main) and
  compare screen-by-screen.
- **Implement-first, tests-after** (decision 2026-07-17). Each sub-phase: build the View → reach manual
  parity against `../mindscape-ref` → **then** lock it in with VM/command unit specs + one axe smoke
  spec per page. Not TDD; the parity bar is the running `main`, and the decks pages have almost no RTL
  tests to port (only `DeckContentEditor.test.tsx` exists on `main`).
- **`start()` is never called from a component.** `main` calls `store.getState().start()` in page
  `useEffect`s; **that block is dropped on every port** — our stores are started once at the composition
  root (`src/composition-root.ts`).
- **No manual `z-index` on shadcn/Base UI overlays.** The hand-rolled nav and status-bar cap use
  `--ms-z-nav`; `main`'s literal `z-[200]` / `z-1000000000` / `z-50` warts do not come across.
- **Every drawer is the shared `Drawer`** (`src/shared/ui/drawer.tsx`, `VirtualKeyboardProvider` baked
  in — Task 1a.4). Ported `*Sheet` symbols rename to `*Drawer` (glossary word is Drawer).
- **The token scale is rem.** `max-w-[430px]` → `max-w-[26.875rem]`; `text-[length:var(--p-text-*)]`
  → `--ms-text-*`; never a px font size.
- **Per-phase gate:** `npm run typecheck && npm run lint && npm run test && npm run build` green at each
  sub-phase close (P1a, P1b, P1c). Intermediate commits may be red (no CI; refactor quality over
  continuous buildability). Husky + lint-staged run Prettier on commit.
- **Device checks are explicit DoD items but do not block committing code.** iOS Drawer feel,
  keyboard-inset scrolling, and the `theme-color`/status-bar cap are verified on a real device (the iOS
  simulator misrenders them); the plan flags each, the learner runs them.

### Task 1a.5: `OverlayHost` + imperative `openDrawer` / `openDialog`

The load-bearing overlay seam (A.4). A sheet that returns data exposes a co-located promise-returning
`open*()` (`const result = await openMoveDeckDrawer(data)`). This host mounts an arbitrary component into
a portal and owns the promise; it resolves when the component calls its injected `resolve`. It is **not**
a Middle Man — it does real work (mount, own the promise, resolve on dismiss).

**Files:**

- Create: `src/shared/ui/overlay-host.tsx`
- Modify: `src/shared/ui/index.ts`, `src/app.tsx` (mount `<OverlayHost/>`)
- Test: `src/shared/ui/overlay-host.spec.tsx`

**Interfaces:**

- Produces:
  - `OverlayHost()` — a component mounted once at app root
  - `openOverlay<R>(render: (resolve: (value: R) => void) => ReactNode): Promise<R>` — low-level
  - `type OverlayResolver<R> = (value: R) => void`
- Consumes: nothing app-specific

- [ ] **Step 1: Write `overlay-host.tsx`.** An external mutable registry (module-level) plus a subscribed
      host. `openOverlay` pushes an entry and returns a promise; the entry's `resolve` both settles the
      promise and unmounts the entry.

```tsx
import { type ReactNode, useSyncExternalStore } from 'react'

export type OverlayResolver<R> = (value: R) => void

interface Entry {
  id: number
  node: ReactNode
}

let entries: Entry[] = []
const listeners = new Set<() => void>()
let nextId = 1

function emit() {
  for (const l of [...listeners]) l()
}

/**
 * Mounts `render(resolve)` into the OverlayHost and returns a promise that settles when the
 * mounted component calls `resolve`. The entry unmounts on resolve. Generic over the result type
 * `R`, so `openConfirmDialog` resolves `boolean`, `openMoveDeckDrawer` resolves a destination, etc.
 */
export function openOverlay<R>(render: (resolve: OverlayResolver<R>) => ReactNode): Promise<R> {
  return new Promise<R>((resolvePromise) => {
    const id = nextId++
    const resolve: OverlayResolver<R> = (value) => {
      entries = entries.filter((e) => e.id !== id)
      emit()
      resolvePromise(value)
    }
    entries = [...entries, { id, node: render(resolve) }]
    emit()
  })
}

export function OverlayHost() {
  const snapshot = useSyncExternalStore(
    (cb) => {
      listeners.add(cb)
      return () => listeners.delete(cb)
    },
    () => entries,
    () => entries,
  )
  return (
    <>
      {snapshot.map((e) => (
        <div key={e.id}>{e.node}</div>
      ))}
    </>
  )
}
```

> Base UI's `Drawer`/`Dialog` manage their own open/close/animation state, so the host does **not**
> hand-roll a state machine — each mounted overlay is a controlled Base UI component that starts open
> and calls `resolve` on confirm or dismiss. This is why the host is ~40 lines, not the ~80 the parent
> plan budgeted against the PrimeNG bridge.

- [ ] **Step 2: Mount `<OverlayHost/>`** in `src/app.tsx`, as a sibling of `<RouterProvider>` inside
      `<ServicesProvider>` (so overlays can `useServices()`), before `<Toaster/>`.

function syncThemeColor(resolved: 'light' | 'dark') {

```tsx
import { type ReactNode, useEffect } from 'react'

/**
 * Publishes `--kb-inset`: the pixels of viewport the on-screen keyboard overlaps. AppScreen consumes
 * it as bottom scroll-padding (ADR-0010 amendment) so every page gains exactly keyboard-height of
 * scroll room while typing — controls under the keyboard are reached by scrolling, never by lifting.
 */
export function KeyboardInsetProvider({ children }: { children: ReactNode }) {
  useEffect(() => {
    const vv = window.visualViewport
    if (!vv) return
    const root = document.documentElement
    const update = () => {
      const inset = Math.max(0, window.innerHeight - vv.height - vv.offsetTop)
      root.style.setProperty('--kb-inset', `${inset}px`)
    }
    update()
    vv.addEventListener('resize', update)
    vv.addEventListener('scroll', update)
    return () => {
      vv.removeEventListener('resize', update)
      vv.removeEventListener('scroll', update)
      root.style.removeProperty('--kb-inset')
    }
  }, [])
  return children
}
```
